# SDLC ROADMAP

## 1. Purpose

Dokumen ini mendefinisikan roadmap pengembangan Barbershop CRM berdasarkan pendekatan Software Development Life Cycle (SDLC) mengacu pada ISO/IEC/IEEE 12207.

---

# 2. Project Roadmap Overview

```text
Planning
    ↓
Analysis
    ↓
Process Design
    ↓
System Design
    ↓
Development
    ↓
Testing
    ↓
Deployment
    ↓
Operations & Maintenance
```

---

# 3. Phase 1 — Planning

## Objective

Menentukan tujuan proyek, ruang lingkup, visi, misi, dan backlog awal.

## Deliverables

* Project Charter
* Vision & Mission
* Scope Definition
* SDLC Roadmap
* Product Backlog

## Status

Completed

---

# 4. Phase 2 — Analysis

## Objective

Mendefinisikan kebutuhan bisnis dan kebutuhan sistem.

## Deliverables

* Business Requirements
* Functional Requirements
* Non Functional Requirements
* Use Case List
* User Stories

## Output

Business Requirement Specification (BRS)

---

# 5. Phase 3 — Process Analysis

## Objective

Memodelkan proses bisnis yang akan diotomatisasi.

## Deliverables

* SIPOC
* BPMN Customer
* BPMN Service
* BPMN Booking
* Process Matrix

## Output

Business Process Specification

---

# 6. Phase 4 — System Design

## Objective

Membuat blueprint teknis sistem.

## Deliverables

* System Architecture
* Data Model
* ERD
* Screen Flow
* Component Design
* API Design

## Output

System Design Specification

---

# 7. Phase 5 — Development

## Objective

Mengimplementasikan desain menjadi aplikasi.

## Deliverables

### Dashboard Module

* Dashboard UI
* KPI Summary

### Customer Module

* Customer CRUD
* Search Customer
* Customer History

### Service Module

* Service CRUD

### Booking Module

* Booking CRUD
* Booking Status Management

---

# 8. Phase 6 — Testing

## Objective

Memastikan sistem berjalan sesuai requirement.

## Deliverables

* Test Strategy
* Test Scenario
* Test Cases
* UAT
* Defect Log

---

# 9. Phase 7 — Deployment

## Objective

Menyiapkan sistem untuk digunakan.

## Deliverables

* Deployment Plan
* Environment Setup
* Release Checklist
* Maintenance Plan

---

# 10. Phase 8 — Operations & Maintenance

## Objective

Menjaga sistem tetap berjalan dan berkembang.

## Deliverables

* Incident Log
* Change Request
* Release Notes
* Version History
